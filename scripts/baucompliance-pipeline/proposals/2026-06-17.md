## Proposal S — Announce Vault archive/restore pending state

- **Problem:** Vault archive/restore actions disable controls while a mutation is pending, but screen-reader users do not get an explicit live pending announcement.
- **Scope:** Add `role="status"` / `aria-live="polite"` pending text and `aria-busy` semantics to the affected Vault card; cover with a focused RTL test.
- **Expected impact:** Better accessibility and confidence during document-workflow mutations.
- **Risk:** 2/10
- **Effort/size:** S

## Proposal M — Add Tech Vault to shared dashboard navigation

- **Problem:** `/dashboard/vault` is a first-class product surface for photo evidence and project documents, but the shared sidebar/mobile dashboard nav only exposes Handover, Deadlines, Cases, and Settings. Users can land in Vault via contextual handoffs but cannot reliably return to it from global dashboard navigation.
- **Scope:** Add a Vault nav item to the shared `navItems`, add localized `menu-vault` labels for DE/FR/IT/EN, and add focused tests proving desktop and mobile navigation expose the Vault link and active mobile title.
- **Expected impact:** Improves discoverability and cross-workflow continuity between audit, cases, and evidence vault tasks.
- **Risk:** 3/10
- **Effort/size:** M
