# Daily Product Improvement Proposals — 2026-07-14

## User and market lens

Primary target users are Swiss construction owners, contractors, and project administrators who move between a defect case, a handover/protocol workflow, evidence capture in the Technical Vault, and reminder/export actions. The workflow moment with the most leverage today is immediately after a user finalizes a handover protocol linked to an existing case: the product already syncs the linked-case checklist, but the user is dropped into a generic success/download state and has to infer where the updated legal case context lives next.

A best-in-market Swiss construction-compliance product should make the next audit step obvious at the exact moment a legally relevant artifact is finalized. It should not only create a PDF/protocol; it should guide the user back to the case ledger or evidence workspace where that protocol becomes part of readiness, deadlines, and follow-up proof.

## Proposal S — Show a linked-case follow-up action after protocol finalization

- **Target user and job-to-be-done:** Swiss construction project admins who finalize a handover/defect protocol linked to a tracked case and need to continue legal follow-up without losing context.
- **Problem:** After finalization, the success state only offers PDF download and a new protocol. If the protocol was linked to a case, the user does not get a direct cue that the case ledger/checklist was updated or where to review next legal actions.
- **Scope:** On the dashboard protocol success step, when a linked case remains effective, render a compact localized follow-up panel with a link back to the matching case list/search. Reuse existing `buildVaultProjectCasesHref` URL semantics; do not change persistence, PDF generation, deadline math, or checklist sync behavior.
- **Expected impact:** Makes the protocol → case audit loop explicit at the moment users complete a legal artifact.
- **User-visible outcome:** After finalizing a linked protocol, users see a CTA to review the linked case follow-up state instead of only generic download/new actions.
- **Competitive / best-product rationale:** Best-in-market compliance products guide users from artifact creation into an audit-ready ledger. This is a safe phase-1 bridge toward the recurring audit cockpit theme.
- **Risk (1-10):** 2
- **Effort/size:** S
- **Phase-1 slice:** The success-state CTA only; no new activity model, persistence schema, or timeline aggregation.

## Proposal M — Add a dashboard “next legal action” summary for the selected linked case

- **Target user and job-to-be-done:** Users preparing a protocol while linked to a case who need the protocol screen to explain how the protocol affects the case deadline/checklist context.
- **Problem:** The dashboard shows linked-case context in step 1 and final review, but the sidebar compliance preview remains generic and does not aggregate the selected case's next action, checklist readiness, and artifact status.
- **Scope:** Add a read-only sidebar summary that combines selected-case status, next action, deadline countdown, and checklist progress using existing case timeline helpers.
- **Expected impact:** Improves explainability during protocol creation and reduces context switching to Cases/Vault.
- **User-visible outcome:** Users can see why the protocol matters for the linked case before finalizing.
- **Competitive / best-product rationale:** Moves toward a legal command-center experience where each artifact is framed by deadline and readiness context.
- **Risk (1-10):** 5
- **Effort/size:** M
- **Phase-1 slice:** Start with selected-case read-only summary only; defer aggregate activity timelines and persistence changes.

## Proposal L — Build a unified audit-ready activity timeline across cases, protocols, vault evidence, and reminders

- **Target user and job-to-be-done:** Owners/contractors preparing for a dispute, handover, or legal review who need a complete chronological audit trail across every compliance artifact.
- **Problem:** The product has strong individual surfaces, but activity context is scattered across Cases, Protocols, Vault, reminder exports, and dashboard actions.
- **Scope:** Create a shared activity derivation layer and UI timeline that lists case creation, protocol finalization, evidence/vault entries, checklist updates, reminders exported, and deadline status changes.
- **Expected impact:** Strategic step-change toward audit readiness and market differentiation.
- **User-visible outcome:** A single timeline of what happened, what is ready, and what legal action remains.
- **Competitive / best-product rationale:** Swiss construction-compliance users need defensible chronology; a unified audit trail is a clear best-product feature.
- **Risk (1-10):** 8
- **Effort/size:** L
- **Phase-1 slice:** Define a read-only derived activity helper for one case detail/card before broad UI rollout.
