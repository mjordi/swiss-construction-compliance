# Daily Product Improvement Proposals — 2026-07-13

## PM insights

Primary user/job-to-be-done: Swiss construction owners, contractors, and project teams who open BauCompliance.ch to understand whether a defect case has enough legal evidence, protocols, reminders, and next-action clarity to withstand a construction-compliance dispute.

Workflow moment: after recent work confirmed reminder exports and synchronized checklist state, users now need a more audit-ready overview before drilling into individual cases. The Technical Vault is the natural proof/evidence surface, but its project cards still emphasize storage and percentages more than the concrete legal action/status behind the file set.

Best-in-market lens: a Swiss construction-compliance product should make evidence readiness and next legal action obvious wherever project proof is reviewed, not only inside the cases list. The best product would progressively become an audit cockpit that connects cases, protocols, reminders, and evidence without making users assemble context manually.

Recent pipeline pattern: the last 14 days were dominated by user-facing legal workflow reliability: case next actions, reminder exports, protocol final review, vault readiness counts, and checklist synchronization. Most slices were S-sized; the repeatedly backlogged strategic theme is an audit-ready activity/cockpit view. Today should step toward that theme with a bounded visible slice instead of another export-feedback patch.

## S — Surface next legal action on Vault project cards

- **Target user and job-to-be-done:** Swiss owners/contractors reviewing project evidence in the Technical Vault to decide whether the proof set is ready or needs action.
- **Problem:** Vault cards show document counts and readiness percentage, but users cannot see the next legal action/deadline context without opening the cases workflow.
- **Scope:** Add a compact audit-action snapshot to each Vault project card using the existing shared case timeline (`nextAction`, countdown/deadline label) plus current protocol/checklist counts.
- **Expected impact:** Users reviewing evidence get immediate legal context and can decide whether to open the case, create a protocol, or complete reminders.
- **User-visible outcome:** Each Vault card names the next legal action and deadline/countdown context alongside proof/readiness indicators.
- **Competitive / best-product rationale:** Moves the Vault from passive file storage toward an audit-ready construction-compliance cockpit.
- **Risk (1-10):** 2 — display-only, built from existing timeline/card data.
- **Effort/size:** S.
- **Phase-1 slice:** This proposal is already the phase-1 slice: read-only card summary only, no persistence, no new route.

## M — Add a first audit-activity summary card to Vault project rows

- **Target user and job-to-be-done:** Project admins checking whether each project has case, protocol, evidence, and reminder artifacts ready for dispute/audit review.
- **Problem:** Artifact state is spread across cases, protocols, vault, and reminders. Vault cards could become the first consolidated proof summary, but a full activity contract needs careful copy and data modeling.
- **Scope:** Add a richer read-only activity section per Vault project with protocol count, checklist completion, reminder export state, evidence attached state, and next legal action, with localized labels and stronger grouping.
- **Expected impact:** Stronger review confidence and a clear stepping stone toward an audit cockpit.
- **User-visible outcome:** Users see a mini activity ledger on each Vault project card.
- **Competitive / best-product rationale:** Best-in-market tools make legal proof status explainable at glance rather than hiding it in separate workflows.
- **Risk (1-10):** 5 — broader copy/layout and data-contract work, likely touches multiple tests.
- **Effort/size:** M.
- **Phase-1 slice:** Start with the S proposal's compact next-action snapshot, then expand to multiple artifact states in a later PR.

## L — Build a unified audit-ready activity timeline across cases, protocols, vault evidence, and reminders

- **Target user and job-to-be-done:** Construction teams preparing or defending compliance decisions who need a chronological, audit-ready proof trail.
- **Problem:** The product has many useful legal artifact actions, but no single timeline that explains what happened, what proof exists, and what remains legally risky.
- **Scope:** Define a shared event model from cases, protocols, vault artifacts, reminders, and checklist state; render a chronological project/case activity timeline; support filtering and export/review affordances.
- **Expected impact:** Major product differentiation and higher trust for legal-dispute preparation.
- **User-visible outcome:** A unified audit trail showing defect discovery, deadlines, protocol finalization, evidence, reminders, and next actions.
- **Competitive / best-product rationale:** A compliance product should not just store documents; it should prove readiness and sequence decisions.
- **Risk (1-10):** 8 — broad data-model, UX, and persistence implications.
- **Effort/size:** L.
- **Phase-1 slice:** A read-only derived timeline for one selected case/project using existing confirmed state, after card summaries establish the copy contract.
