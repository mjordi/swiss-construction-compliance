# Daily Product Improvement Proposals — 2026-07-08

## User and market lens

Primary users are Swiss construction owners, contractors, and project admins using BauCompliance to turn defect discovery into audit-ready follow-up: notice drafting, evidence capture, protocol finalization, deadline reminders, and proof that required steps were completed. The workflow moment with the most risk today is the case follow-up checklist: users rely on checklist state as their legal readiness ledger, so a control that can accidentally undo a completed calendar-reminder step creates legal-trust risk.

A best-in-market Swiss construction-compliance product should make reminders, evidence, and notice-readiness deterministic and audit-friendly: repeating an export should reinforce completion, not toggle it off; dashboard summaries should eventually connect cases, protocols, vault evidence, and deadlines into a single activity cockpit.

Recent 14-day pipeline context shows strong user-facing work around deadline reminders, cases, vault readiness, and protocol finalization. The pattern is mostly S-sized, with several bounded M slices. Today should still consider a strategic step-up, but the safest high-leverage slice is a reliability fix on a legal checklist action rather than another cosmetic label.

## Proposal S — Make case calendar reminder exports idempotent

- **Target user and job-to-be-done:** Swiss construction admins exporting a 60-day notice `.ics` from a case and using the follow-up checklist as proof that reminder setup was completed.
- **Problem:** The current export action calls the same toggle helper used by manual checklist checkboxes. If `calendarReminderExported` is already complete and the user downloads the `.ics` again, the action can flip the checklist item back to incomplete, undermining readiness trust.
- **Scope:** Replace export-triggered checklist toggling with an explicit set-to-complete update, while preserving manual checkbox toggles. Add focused regression coverage proving repeated export leaves the calendar checklist item complete and persists `calendarReminderExported: true`.
- **Expected impact:** Prevents a legal readiness marker from being accidentally undone by a safe repeat action.
- **User-visible outcome:** Users can re-download a case reminder without the checklist or reminder-readiness state regressing.
- **Competitive / best-product rationale:** Audit/readiness workflows should be idempotent for artifact exports; repeated downloads should not alter legal state negatively.
- **Risk:** 2/10 — narrow state-management change on an existing action.
- **Effort/size:** S.
- **Phase-1 slice:** Implement the idempotent set-to-complete path for the case `.ics` export only.

## Proposal M — Add a case row “last exported reminder” confirmation state

- **Target user and job-to-be-done:** Project admins reviewing whether a deadline reminder was actually generated after exporting a case `.ics`.
- **Problem:** Exporting a reminder currently updates checklist readiness but does not provide a durable, explicit confirmation near the export action beyond checklist state.
- **Scope:** Add localized inline success/error feedback next to the export action, timestamp/copy when available, and tests for stale completion and persistence failure.
- **Expected impact:** Improves confidence at the exact reminder-export moment.
- **User-visible outcome:** A case row clearly confirms the reminder export action and whether checklist persistence succeeded.
- **Competitive / best-product rationale:** Best legal workflow tools show a clear audit trail for artifact creation steps.
- **Risk:** 4/10 — touches async feedback and checklist persistence UI.
- **Effort/size:** M.
- **Phase-1 slice:** First ship idempotent export state; later add richer confirmation if still needed.

## Proposal L — Unified compliance activity cockpit across cases, protocols, vault evidence, and reminders

- **Target user and job-to-be-done:** Owners and contractors needing one place to see which project needs legal action next across deadlines, evidence, protocols, and reminders.
- **Problem:** Recent improvements have enriched individual surfaces, but users still need to synthesize readiness from cases, vault cards, protocol finalization, and deadline exports.
- **Scope:** A new read-only cockpit route/card that aggregates next legal action, evidence readiness, protocol count/finalization state, reminder readiness, and unresolved failures.
- **Expected impact:** Major reduction in cognitive load for compliance triage.
- **User-visible outcome:** Users land on one ranked action list rather than scanning multiple pages.
- **Competitive / best-product rationale:** A market-leading Swiss compliance product should make the next legally relevant action obvious and auditable.
- **Risk:** 8/10 — broad data contract, copy, and routing work.
- **Effort/size:** L / strategic.
- **Phase-1 slice:** After idempotent artifact actions and feedback contracts are stable, add a read-only activity summary card fed by existing case/vault/protocol counts.

- Cases and deadline reminders have been touched recently, so the approved slice should avoid new broad behavior and focus on a concrete legal-state correctness issue.
- The S proposal is small but not cosmetic: it prevents a repeated export from corrupting the visible readiness ledger users rely on before notice deadlines.
