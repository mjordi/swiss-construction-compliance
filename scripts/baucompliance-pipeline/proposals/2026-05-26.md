## Proposal S — Harden Settings success feedback against stale timers
- **Title:** Harden Settings success feedback against stale timers
- **Problem:** `app/dashboard/settings/page.tsx` uses raw `setTimeout` calls to clear profile-save and password-update success states, but it does not cancel or invalidate those timers when the user edits again or the component unmounts. That can re-show stale success feedback after the form state has already changed.
- **Scope:** Add timer cleanup/invalidation for profile and password success feedback, clear stale success state when inputs change, and add focused regression tests for success reset, edit-after-save behavior, and unmount cleanup.
- **Expected impact:** Improves trust in account settings feedback with a low-risk reliability patch.
- **Risk (1-10):** 3
- **Effort/size:** Small

## Proposal M — Add archive and restore actions to Vault projects
- **Title:** Add archive and restore actions to Vault projects
- **Problem:** The Vault already exposes active and archived tabs, and the case model already supports `archived`, but users still have no UI action to move a project between those states. That makes the Archived tab mostly passive and prevents firms from cleaning up completed matters without leaving the Vault.
- **Scope:** Add archive and restore controls on Vault project cards, persist the status change through Supabase, update local list state immediately so cards move between tabs without a full reload, and add regression tests for successful archive/restore flows plus failure handling.
- **Expected impact:** Turns an existing but incomplete Vault workflow into a usable day-to-day project-lifecycle action, helping users keep active evidence workspaces focused while retaining access to older matters.
- **Risk (1-10):** 5
- **Effort/size:** Medium
