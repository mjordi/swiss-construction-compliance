## S — Confirm public Ruegefrist calendar downloads

- **Target user and job-to-be-done:** A homeowner, contractor, or admin using the public free Ruegefrist calculator to turn a calculated 60-day deadline into a calendar reminder.
- **Problem:** The public calculator's `.ics` download action is silent. If browser blob creation fails or the click is blocked, the user receives no localized success/error feedback even though the action protects a legal deadline.
- **Scope:** Add temporary localized success/error feedback around `components/ruegefrist-calculator.tsx` calendar export; clear stale feedback when dates, reminder presets, reset, or unmount make the exported artifact stale; add focused tests in `__tests__/ruegefrist-calculator.test.tsx`; add locale keys.
- **Expected impact:** Users gain immediate confidence that the deadline reminder file was prepared, matching the product's dashboard export-feedback standard.
- **User-visible outcome:** The download button temporarily changes to a success or retry/error label after attempting the export.
- **Competitive / best-product rationale:** Deadline exports are core legal-risk artifacts; a best-in-market calculator should not leave users guessing whether a reminder was produced.
- **Risk (1-10):** 2
- **Effort/size:** S
- **Phase-1 slice:** This proposal is already the smallest reviewable slice.

## M — Add a first audit-activity summary card to Vault project rows

- **Target user and job-to-be-done:** Project admins reviewing a project in Tech Vault who need to understand not just readiness percentage, but which recent legal artifact actions support that score.
- **Problem:** Vault cards now show checklist counts and protocol document counts, but not a compact activity/proof summary. Users still need to click through to infer whether reminders, evidence, notices, and protocols have been acted on.
- **Scope:** Add a read-only summary line using existing case checklist/protocol count data (for example latest confirmed artifact categories) without introducing a new persistence model.
- **Expected impact:** Improves audit readiness and decision confidence from the vault overview.
- **User-visible outcome:** Each project card shows a concise "proof chain" snapshot beside readiness percentage.
- **Competitive / best-product rationale:** A leading compliance product should surface audit proof at overview level, not hide it across multiple workflows.
- **Risk (1-10):** 5
- **Effort/size:** M
- **Phase-1 slice:** Render a non-interactive summary from existing normalized checklist fields only; backlog richer timestamps/activity history.

## L — Build a unified audit-ready activity timeline across cases, protocols, vault evidence, and reminders

- **Target user and job-to-be-done:** Owners, contractors, and legal/admin teams who must reconstruct the compliance story for a defect case months later.
- **Problem:** The product currently distributes legal evidence across case rows, protocol finalization state, reminder exports, and vault/project readiness. There is no single audit trail view.
- **Scope:** Define an event model and UI timeline that derives from existing cases, protocols, checklist/readiness fields, exports, and vault evidence, then lets users review and export the chronology.
- **Expected impact:** Major strategic differentiation and legal trust improvement.
- **User-visible outcome:** A project/case activity timeline showing deadline calculation, reminder export, protocol finalization, evidence capture, and notice readiness.
- **Competitive / best-product rationale:** Best-in-market compliance tools provide explainable proof trails, not only isolated task completion controls.
- **Risk (1-10):** 8
- **Effort/size:** L
- **Phase-1 slice:** Start with a read-only derived timeline for one case from current checklist/protocol data, no new persistence or export schema.
