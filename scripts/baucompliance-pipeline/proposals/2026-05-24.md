## Proposal S — Harden Settings success feedback against stale timers
- **Title:** Harden Settings success feedback against stale timers
- **Problem:** `/app/dashboard/settings/page.tsx` sets transient success banners for profile and password updates with raw `setTimeout` calls, but it does not clear or invalidate those timers when the user edits again or the component unmounts. That can surface stale success state after the underlying form has already changed.
- **Scope:** Add timer/request cleanup for profile/password success feedback, clear stale success states when inputs change or the component unmounts, and add focused regression tests.
- **Expected impact:** Improves trust in account-management feedback with a very small, low-risk reliability patch.
- **Risk (1-10):** 3
- **Effort/size:** Small

## Proposal M — Persist deadline reminder presets in shared links
- **Title:** Persist deadline reminder presets in shared links
- **Problem:** `/dashboard/deadlines` lets users customize reminder offsets for the exported ICS, but copied/shared links only preserve `acceptance=`. When another teammate opens the link, the page silently falls back to the default reminder set, so the shared view no longer matches what the sender configured.
- **Scope:** Add sanitized reminder query-param support on `/dashboard/deadlines`, include the selected reminder offsets in copied/shared links and calculated URLs, restore them on hydration, and add focused regression tests for valid/invalid reminder state.
- **Expected impact:** Makes a collaboration-oriented compliance workflow more trustworthy by ensuring shared deadline setups reproduce the same reminder plan instead of an implicit default.
- **Risk (1-10):** 4
- **Effort/size:** Medium
