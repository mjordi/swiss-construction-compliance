## Proposal S — Keep case dashboard filters in sync with shared URLs
- **Title**: Keep case dashboard filters in sync with shared URLs
- **Problem**: The cases page writes `regime`, `status`, `sort`, and `q` into the URL, but it does not re-hydrate local state when those query params change later. Users can land on a filtered link or navigate with browser history and see stale controls/results.
- **Scope**: Add a narrow URL-to-state synchronization layer for the cases dashboard filters/search, preserve the existing URL-writing behavior, and add focused regression tests covering initial restore plus external query-param changes.
- **Expected impact**: Makes shared filter links and browser navigation trustworthy in a core follow-up workflow without expanding product scope.
- **Risk (1-10)**: 3
- **Effort/size**: Small

## Proposal M — Add a copy-link action for filtered case triage views
- **Title**: Add a copy-link action for filtered case triage views
- **Problem**: The cases page already encodes filters in the URL, but there is no explicit affordance for teams to copy a triage view and share it with collaborators.
- **Scope**: Add a copy-link CTA near the filters, localize copied-state feedback, and make the shared link restore the same case filter/search configuration.
- **Expected impact**: Improves collaboration around urgent case review and turns the existing URL-state behavior into an obvious user-facing feature.
- **Risk (1-10)**: 4
- **Effort/size**: Medium

## Proposal L — Add per-case detail drill-down with linked protocol and evidence context
- **Title**: Add per-case detail drill-down with linked protocol and evidence context
- **Problem**: Users still need to move between the cases dashboard, protocol flow, and vault to understand the full context for a single case.
- **Scope**: Add a richer case detail surface that combines deadline status, checklist progress, linked protocols, and evidence context into one view.
- **Expected impact**: Deepens product usefulness for active compliance follow-up and reduces cross-page context switching.
- **Risk (1-10)**: 7
- **Effort/size**: Large
