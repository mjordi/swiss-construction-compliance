# Daily Product Improvement Proposals — 2026-07-01

## Research lens

**Primary user / job-to-be-done:** Swiss construction owners, contractors, and project admins who move from a tracked defect/deadline case into protocol finalization and need the protocol workflow to preserve the legally relevant next action without making them re-interpret dates.

**Workflow moment where trust is lost:** After creating or opening a compliance case, the user starts a defect/protocol record from that case. The dashboard currently acknowledges the linked case and shows the deadline date, but the protocol author still has to infer the legal action to take from status/date context instead of seeing the same case-timeline guidance used in the cases dashboard.

**Best-in-market expectation:** A leading Swiss construction-compliance product should make the legally relevant next action obvious at each workflow handoff: from public calculator to case, from case to evidence vault, and from case to protocol. It should not strand legal context on only one page.

**Recent pipeline pattern:** The last 14 days heavily emphasized S-sized reliability/accessibility and URL-state trust patches. Recent strategic slices include cases/vault URL canonicalization and the 2026-06-28 public Ruegefrist-to-cases handoff. A recurring strategic theme has been cross-workflow case activity/timeline context; the pure timeline helper now exists in `lib/case-timeline.ts` and is already consumed by cases and vault, so a narrow dashboard consumption slice is safer than starting another broad timeline workspace.

## Proposal S — Add next-action context to linked cases in the protocol wizard

- **Target user and job-to-be-done:** Project admins or site teams opening `/dashboard?case=<id>` from a case row to create a protocol, needing to record the defect with the right legal follow-up in mind.
- **Problem:** The wizard displays a linked-case status/date chip, but not the concrete next legal action (for example, draft notice / document delivery / immediate notice). Users must mentally translate status into action while authoring evidence and protocol text.
- **Scope:** Reuse the existing case-timeline view model for the selected case in `app/dashboard/page.tsx`, and show the linked case's `nextAction` plus countdown/deadline context in the existing linked-case card. Add focused dashboard regression coverage.
- **Expected impact:** Improves legal clarity at the exact handoff from case tracking into protocol/evidence work without adding new persistence or navigation.
- **User-visible outcome:** When a case is selected or opened from a case link, the dashboard linked-case panel shows the next legal action and deadline/countdown context before the user finalizes the protocol.
- **Competitive / best-product rationale:** Best-in-market compliance tools carry audit/legal context across workflows instead of forcing users to re-derive next steps on each page.
- **Risk (1-10):** 3
- **Effort/size:** S / bounded phase-1 strategic slice

## Proposal M — Show a case activity snapshot across case, protocol, evidence, and reminder readiness

- **Target user and job-to-be-done:** Project teams reviewing a case before a meeting or legal notice, needing one compact view of what is complete and what still needs action.
- **Problem:** Case status, protocol count, vault evidence, checklist, and reminder export state are spread across cases and vault surfaces. Users lose confidence about audit readiness because the product lacks a single snapshot.
- **Scope:** Add a read-only activity snapshot component on the cases page for each row using existing `protocolCounts`, checklist progress, deadline status, and vault links. Keep it non-persistent and avoid a new timeline UI framework.
- **Expected impact:** Medium workflow value: teams can see readiness at a glance before sending notices or exporting protocols.
- **User-visible outcome:** Each case row summarizes legal status, checklist completion, protocol count, evidence/vault path, and reminder readiness.
- **Competitive / best-product rationale:** A premium compliance product should make audit readiness obvious without requiring users to open multiple modules.
- **Risk (1-10):** 5
- **Effort/size:** M
- **Phase-1 slice:** Start with a read-only snapshot inside cases rows; no new persistence, no dashboard home feed, no cross-page timeline route.

## Proposal L — Unified compliance activity timeline and guided notice workspace

- **Target user and job-to-be-done:** Owners/contractors managing multiple legally sensitive defects who need an audit-ready history and guided next step from defect discovery to notice/protocol/evidence/reminder.
- **Problem:** BauCompliance has strong pieces (cases, protocols, vault, reminders), but the user still navigates modules rather than a unified timeline of what happened, what is legally due, and what evidence supports it.
- **Scope:** Create a unified timeline/workspace backed by the case-timeline model and protocol/vault/checklist/reminder events, with guided next actions and exportable audit trail.
- **Expected impact:** High strategic differentiation: turns BauCompliance from a set of tools into a compliance command center.
- **User-visible outcome:** A case detail page or workspace with chronological evidence, protocol, reminder, and legal-action state.
- **Competitive / best-product rationale:** The market leader should provide an audit-ready chain of custody and next-step guidance, not merely calculators and separate lists.
- **Risk (1-10):** 8
- **Effort/size:** L
- **Phase-1 slice:** Extend the existing timeline model with protocol/vault/reminder event fixtures, then expose a read-only timeline for a single selected case before adding editing or exports.
