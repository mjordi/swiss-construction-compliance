## Proposal S — Surface linked-case loading failures in the audit wizard
- **Title**: Surface linked-case loading failures in the audit wizard
- **Problem**: The dashboard protocol wizard silently degrades when linked cases fail to load, so users do not know whether the selector is empty because they have no cases or because loading failed.
- **Scope**: Add localized inline linked-case load feedback and a retry action around the protocol wizard's case selector while preserving the current stale-draft handling and standalone-protocol flow.
- **Expected impact**: Improves trust during protocol creation by making load failures visible and recoverable on a high-value workflow.
- **Risk (1-10)**: 3
- **Effort/size**: Small

## Proposal M — Add a retryable aggregate-load recovery state to the Tech Vault overview
- **Title**: Add a retryable aggregate-load recovery state to the Tech Vault overview
- **Problem**: When the Vault's projects/cases/protocols aggregate load fails, users see a blocking error card but no direct way to retry without a full page refresh.
- **Scope**: Introduce a localized retry control and tighten load-state transitions for the Vault overview queries, with focused regression coverage.
- **Expected impact**: Reduces dead ends on the evidence workflow and aligns error recovery behavior across dashboard surfaces.
- **Risk (1-10)**: 4
- **Effort/size**: Medium

## Proposal L — Build a unified case workspace across timeline, audit, and vault workflows
- **Title**: Build a unified case workspace across timeline, audit, and vault workflows
