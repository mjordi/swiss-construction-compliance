## Proposal S — Add a first-project CTA to the Vault empty state
- **Title:** Add a first-project CTA to the Vault empty state
- **Problem:** When Tech Vault has no active or archived projects, the empty-state body tells users to create a project, but the empty-state card itself renders no action. This makes first-use onboarding weaker than the rest of the Vault flow and forces users to look back to the header.
- **Scope:** Extend the Vault empty-state model with a `create-project` action for the no-content projects state, render it as a real `Link` to the existing project creation destination (`/dashboard`), and add focused tests for the helper and rendered empty-state CTA.
- **Expected impact:** Improves first-run activation for evidence storage with a clear, contextual next step and no new backend or data-model risk.
- **Risk (1-10):** 2
- **Effort/size:** Small

## Proposal M — Add case-aware deadline calculator handoffs
- **Title:** Add case-aware deadline calculator handoffs
- **Problem:** Case cards expose deadline status and `.ics` export, but users who want to inspect or share the broader deadline calculator must manually navigate to the deadlines page and re-enter dates.
- **Scope:** Add a tested helper and card-level link that preloads the deadline calculator from a case date where semantically valid, while preserving existing case filters and avoiding misleading old-law flows.
- **Expected impact:** Better cross-workflow continuity between case triage and the standalone legal deadline tool.
- **Risk (1-10):** 5
- **Effort/size:** Medium

## Proposal L — Build a unified case workspace route
- **Title:** Build a unified case workspace route
- **Problem:** Users still jump between Cases, Protocol Wizard, Vault, and Deadlines to understand the status of one construction claim.
- **Scope:** Introduce a dedicated per-case workspace route that consolidates checklist readiness, protocols, evidence, deadline reminders, and next actions behind a single URL.
- **Expected impact:** A stronger strategic workflow backbone for Swiss construction compliance operations.
- **Risk (1-10):** 8
- **Effort/size:** Large / strategic
