## Proposal S — Localize login auth feedback and shell copy
- **Title**: Localize login auth feedback and shell copy
- **Problem**: The login page still mixes translated headings with English-only placeholders, demo-account CTA text, attribution labels, and raw auth/config failure messages, which weakens trust for DE/FR/IT users during account creation and sign-in.
- **Scope**: Add locale keys for the remaining login shell strings, introduce a small auth-feedback helper that maps common signup/login/config errors to translation keys while preserving unknown provider messages, and cover the helper plus locale keys with focused tests.
- **Expected impact**: Makes the first-use auth flow feel product-complete across supported languages and reduces confusion when authentication actions fail.
- **Risk (1-10)**: 3
- **Effort/size**: Small

## Proposal M — Add password-reset request flow from the login screen
- **Title**: Add password-reset request flow from the login screen
- **Problem**: Users who forget their password have no self-serve recovery path in the current auth entrypoint, which can stall account access and support follow-up.
- **Scope**: Add a password-reset CTA and request state to the login screen, wire it to Supabase password reset, localize the new states, and extend the placeholder auth adapter for build-safe behavior.
- **Expected impact**: Reduces login friction for returning teams and fills an obvious gap in the account access workflow.
- **Risk (1-10)**: 5
- **Effort/size**: Medium

## Proposal L — Add per-project vault drill-down with evidence timeline
- **Title**: Add per-project vault drill-down with evidence timeline
- **Problem**: Technical Vault users still cannot inspect a single project’s evidence, linked protocols, and compliance history without jumping between screens.
- **Scope**: Introduce a project detail view that aggregates linked cases, protocol history, evidence counts, and follow-up guidance from the vault list.
- **Expected impact**: Significantly improves audit readiness and evidence review depth for larger teams managing multiple active disputes.
- **Risk (1-10)**: 7
- **Effort/size**: Large
