# Daily Product Improvement Proposals — 2026-05-30

## PM insights

Recent first-parent history shows a sustained cadence of small, reviewable UI reliability and workflow-handoff improvements: cases inline edit/delete safety, deadlines share links, vault/cases/dashboard handoffs, vault archive actions, and dashboard draft persistence. The strongest daily opportunity is a focused reliability/accessibility fix on the public Rügefrist calculator: its calculated result can remain visible after date inputs change, and its date controls are not directly accessible by label.

## S — Clear stale Rügefrist results when calculator inputs change

- **Problem:** The public Rügefrist calculator keeps showing the last calculated deadlines after the user edits the contract/discovery dates. This can make the visible result and calendar export feel detached from the current inputs. The date inputs and icon-only reset control also lack explicit accessible names.
- **Scope:** Add explicit accessible names to the calculator date/reset/download controls, clear any displayed result when either date input changes, and add focused RTL coverage for the stale-result behavior.
- **Expected impact:** Users no longer act on stale deadline results after editing dates, and the calculator is easier to operate via assistive technology/tests.
- **Risk:** 2/10 — localized to one public calculator component with straightforward regression tests.
- **Effort/size:** S

## M — Add a Rügefrist share link for calculated dates

- **Problem:** The dashboard deadlines page supports shareable URL state, but the public calculator cannot be linked to with prefilled contract/discovery dates.
- **Scope:** Add query-param hydration/sanitization for the public calculator, a copy-link action, and tests for invalid/valid date state.
- **Expected impact:** Advisors and construction teams can share calculator scenarios directly with colleagues or clients.
- **Risk:** 5/10 — touches public URL state, sanitization, copy feedback, and i18n copy.
- **Effort/size:** M

## L — Guided public-to-dashboard defect workflow

- **Problem:** The public calculator helps users understand deadlines, but it does not guide high-risk cases into a saved dashboard case/protocol workflow.
- **Scope:** Design a multi-step handoff from public deadline calculation to authenticated case creation with prefilled dates, onboarding prompts, and saved reminders.
- **Expected impact:** Strategic conversion and product activation path from legal utility to core SaaS workflow.
- **Risk:** 8/10 — broader product funnel, auth, persisted data, and onboarding implications.
- **Effort/size:** L
