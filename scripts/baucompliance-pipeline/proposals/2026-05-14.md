## Proposal S — Add visible initial-load error handling and retry to the cases dashboard
- **Title**: Add visible initial-load error handling and retry to the cases dashboard
- **Problem**: When the initial cases/protocol fetch fails, the page can look blank or stale without telling the user whether data failed to load or there are simply no cases.
- **Scope**: Add a localized load-error banner/state to `/dashboard/cases` with a retry action wired to the existing refresh path, while preserving the current empty-state and filter behavior when loads succeed.
- **Expected impact**: Improves trust on a core compliance workflow and gives users a direct recovery path instead of forcing a manual refresh.
- **Risk (1-10)**: 3
- **Effort/size**: Small
