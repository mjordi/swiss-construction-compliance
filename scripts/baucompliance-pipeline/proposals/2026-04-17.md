# Daily Product Pipeline Proposals — 2026-04-17

## Research phase

### Proposal S — Case card quick-copy for legal deadline line
- **Problem:** Users often need to copy deadline data into email/chat manually.
- **Scope:** Add a one-click copy action on each case card for `project + canton + notice deadline + status`.
- **Expected impact:** Faster case handoff and fewer manual formatting mistakes.
- **Risk (1-10):** 3
- **Effort:** Small

### Proposal M — Segmented status KPI filters in Cases
- **Problem:** Status filter is hidden in select menus; users miss quick triage paths.
- **Scope:** Add dedicated KPI cards for On track, Attention, Urgent, Expired with one-click toggle filters.
- **Expected impact:** Faster status-based review and fewer clicks during daily triage.
- **Risk (1-10):** 6
- **Effort:** Medium

### Proposal L — Team assignment and ownership lanes for cases
- **Problem:** Multi-user teams cannot easily divide responsibility by case owner.
- **Scope:** Add case ownership model (DB + UI), owner filter, and lane views on Cases page.
- **Expected impact:** Better accountability and reduced duplicate follow-up in teams.
- **Risk (1-10):** 8
- **Effort:** Large

## PM insights (3)
1. Cases workflow gains most when triage can happen in one screen with minimal menu friction.
2. Existing status logic is robust, so UI discoverability improvements are lower-risk than new legal-calculation features.
3. Team collaboration features are strategic, but should follow after proving stronger single-user triage velocity gains.
