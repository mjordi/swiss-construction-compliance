## Small (S): Consume dashboard case handoff query after hydration

- **Problem:** The cases page can hand users to the protocol wizard with `/dashboard?case=<id>`. The dashboard hydrates that case into wizard state, but the query parameter remains in the URL. If the user later unlinks/changes the case or refreshes, the stale query can re-apply the old handoff and override the current draft intent.
- **Scope:** After the dashboard successfully resolves a requested case from the URL, remove the consumed `case` query parameter with a non-scrolling route replace while preserving any unrelated query parameters. Add a focused regression test.
- **Expected impact:** Makes the case-to-protocol handoff one-time and prevents stale URL state from surprising users during protocol drafting.
- **Risk:** 2/10
- **Effort/size:** Small

- **Risk:** 5/10
- **Effort/size:** Medium
