## Proposal S — Harden account settings save feedback
- **Title**: Harden account settings save feedback
- **Problem**: The settings page silently ignores profile load/save failures and always presents the profile action as if saving succeeded, which can make teams think account details were updated when persistence actually failed.
- **Scope**: Add explicit profile load/save error handling, track the last loaded profile snapshot so the save button only activates for unsaved changes, and cover the state helper with focused tests plus locale keys.
- **Expected impact**: Improves trust in a core account workflow, reduces accidental duplicate saves, and gives users clear recovery guidance when profile persistence fails.
- **Risk (1-10)**: 2
- **Effort/size**: Small

## Proposal M — Localize login auth feedback and placeholders
- **Title**: Localize login auth feedback and placeholders
- **Problem**: The login/signup page still exposes several English-only validation, success, and fallback auth messages inside an otherwise multilingual product, weakening first-use clarity for non-English users.
- **Scope**: Move hardcoded login/signup status messages and placeholders to locale keys, normalize common auth fallback messages, and extend locale coverage tests.
- **Expected impact**: Makes the acquisition and sign-in funnel feel product-complete for DE/FR/IT users and reduces confusion when signup or login actions fail.
- **Risk (1-10)**: 3
- **Effort/size**: Medium

## Proposal L — Add a project-level vault drill-down timeline
- **Title**: Add a project-level vault drill-down timeline
- **Problem**: The Technical Vault stops at a card overview, so users cannot inspect a single project’s evidence, linked protocols, and compliance history without manually hopping across screens.
- **Scope**: Introduce a project detail view that aggregates linked cases, protocol history, evidence counts, and next-step guidance from the vault list.
- **Expected impact**: Significantly improves audit readiness and evidence review depth for larger teams managing multiple active disputes.
- **Risk (1-10)**: 7
- **Effort/size**: Large
