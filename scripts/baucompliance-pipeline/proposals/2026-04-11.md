# Research Proposals — 2026-04-11

## PM Insights (from market + repo signal)
1. Swiss public-sector construction and permit processes continue to digitize (eConstruction/eBau tracks), which raises expectations for clean case data and fast operational workflows in contractor tooling.
2. Recent delivery cadence is strongest on small, review-friendly increments, so today should prefer low-risk UX friction removal over broad platform changes.
3. Core workflow depth (cases, deadlines, reminder follow-through) remains the primary strategic moat versus marketing-surface work.

## Proposal S — One-click clear control in case search
- **Size**: Small
- **Problem**: Clearing a case search query currently depends on keyboard behavior (Escape) or manual deletion.
- **User value**: Faster filter reset during triage, especially for mixed keyboard/mouse users.
- **Implementation sketch**: Add inline clear button in search input; keep focus in search after clear.
- **Risk**: Low

## Proposal M — Saved “quick filter presets” for case triage
- **Size**: Medium
- **Problem**: Teams repeatedly re-apply the same status/regime/sort combinations.
- **User value**: One-click return to common triage views (e.g., urgent new-law cases).
- **Implementation sketch**: Add 2-3 local saved presets in cases view using URL-state snapshots.
- **Risk**: Medium

## Proposal L — Team deadline digest and escalation inbox
- **Size**: Large
- **Problem**: Deadline follow-through still depends on individual user checks.
- **User value**: Shared accountability and reduced missed notices.
- **Implementation sketch**: Scheduled digest generation, escalation routing, and audit trail for unresolved urgent cases.
- **Risk**: High
