## Proposal S — Guard case creation against duplicate pending submits

- **Problem:** The case creation form disables its submit button after `saving` state updates, but `handleAddCase` has no synchronous in-flight guard and the form fields/cancel action remain interactive while the insert request is pending. A double submit or mid-flight edit can create confusing outcomes or duplicate rows.
- **Scope:** Add a narrow create-save in-flight ref guard, disable create inputs/cancel while saving, and add a focused regression test that a duplicate submit is ignored and the form controls are locked during the pending request.
- **Expected impact:** Reduces accidental duplicate case creation and prevents user edits/cancel actions from racing a pending Supabase insert.
- **Risk (1-10):** 2 — localized to create form state and tests.
- **Effort/size:** S

## Proposal M — Add a shared pending-mutation guard abstraction for cases

- **Problem:** The cases page now has multiple independently managed pending states (`saving`, `deletingCaseIds`, `updatingCaseId`, `checklistSavingByCase`), increasing the chance that future sibling actions miss a lockout path.
- **Scope:** Extract a small helper or hook for case-row/action pending checks, update create/edit/delete/checklist/export controls to use it consistently, and extend tests across at least two mutation surfaces.
- **Expected impact:** Improves maintainability and makes race prevention more systematic across the most important operational workflow.
- **Risk (1-10):** 5 — touches several interactions in a large page and could cause over-locking.
- **Effort/size:** M

## Proposal L — Build a guided compliance action center across cases, vault, and protocols

- **Problem:** Compliance follow-up currently spans separate cases, evidence vault, protocol generation, and deadline reminder surfaces. Users still need to infer the next best action across those workflows.
- **Scope:** Design and implement a cross-workflow action center that aggregates urgent cases, missing evidence, unsent notices, and protocol completion status with direct CTAs into each workflow.
- **Expected impact:** High strategic value: turns the product from record storage into an operational compliance cockpit for Swiss construction teams.
- **Risk (1-10):** 8 — requires product design, cross-surface data modeling, and careful i18n/accessibility coverage.
- **Effort/size:** L
