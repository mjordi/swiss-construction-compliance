# Research Proposals — 2026-04-28

## PM Insights (market + repo signal)
1. **Delivery signal:** The last 7 days show strong merge throughput (8 merged PRs), mostly reliability and workflow hardening, so today should favor low-risk flow integrity over net-new surface area.
2. **Workflow signal:** Protocol finalization still has a same-tick duplicate-submit window (rapid multi-trigger before state disable settles), which can create avoidable duplicate write attempts.
3. **Compliance signal:** Evidence expectations are shifting from static records to auditable lifecycle traces, keeping protocol event history as a high-value strategic direction but not a day-scale ship.

## Proposal S — Single-flight guard for protocol finalization
- **Size**: Small
- **Problem**: Rapid repeated finalize triggers can race before UI disable fully settles.
- **User value**: Prevents accidental duplicate submit attempts and makes finalization behavior deterministic.
- **Implementation sketch**: Add in-flight single-flight ref guard in finalize handler and lock step-2 interactive controls while save is running.
- **Risk**: Low

## Proposal M — Finalize error-retry state with explicit failure context
- **Size**: Medium
- **Problem**: Save failures surface as a generic alert without contextual retry guidance.
- **User value**: Faster recovery from transient save errors and fewer abandoned protocols.
- **Implementation sketch**: Add localized inline error panel with retry action, preserve step context, and show operation status details.
- **Risk**: Medium

## Proposal L — Protocol lifecycle event timeline (phase 1)
- **Size**: Large
- **Problem**: Users cannot inspect a structured timeline of protocol lifecycle events.
- **User value**: Stronger defensibility and clearer case/protocol auditability.
- **Implementation sketch**: Introduce append-only protocol events model, write on lifecycle transitions, and expose read-only timeline UI.
- **Risk**: High
