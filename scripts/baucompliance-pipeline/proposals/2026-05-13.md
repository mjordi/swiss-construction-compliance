## Proposal S — Add a Vault CTA that opens the matching case follow-up view
- **Title**: Add a Vault CTA that opens the matching case follow-up view
- **Problem**: Vault users can review evidence/project cards, but they still have to manually switch to the cases dashboard and rebuild the relevant search/filter state to continue follow-up work.
- **Scope**: Add a focused CTA on Vault project cards that links into `/dashboard/cases` with a prefilled project query and triage filter for review-status projects, backed by targeted helper/UI tests and any required locale copy.
- **Expected impact**: Reduces friction between evidence review and active compliance follow-up while staying inside one surface plus narrow routing logic.
- **Risk (1-10)**: 3
- **Effort/size**: Small

## Proposal M — Add visible error and retry handling for the cases dashboard initial load
- **Title**: Add visible error and retry handling for the cases dashboard initial load
- **Problem**: When the initial cases/protocol fetch fails, the dashboard can appear empty or stale without explaining whether there are no cases or the load failed.
- **Scope**: Introduce a localized error banner and retry action for the initial cases/protocol refresh path, plus regression coverage for failure and recovery behavior.
- **Expected impact**: Improves trust on a core operational page and prevents users from misreading transient data-load failures as an empty portfolio.
- **Risk (1-10)**: 4
- **Effort/size**: Medium
