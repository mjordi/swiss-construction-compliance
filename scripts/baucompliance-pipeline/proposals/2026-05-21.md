## Proposal S — Add explicit failure feedback for case create/delete actions
- **Title**: Add explicit failure feedback for case create/delete actions
- **Problem**: Creating or deleting a case still awaits Supabase mutations without clearly surfacing failure state, so users can think an action succeeded when it did not.
- **Scope**: Add localized create/delete error feedback to `/dashboard/cases`, preserve the add-case form state on failed creates, and add focused tests for mutation failure handling.
- **Expected impact**: Improves trust and reliability on one core workflow with low implementation risk.
- **Risk (1-10)**: 3
- **Effort/size**: Small

## Proposal M — Add a direct “open in vault” handoff from cases with URL-synced Vault search
- **Title**: Add a direct “open in vault” handoff from cases with URL-synced Vault search
- **Problem**: Users can identify the project that needs evidence follow-up in `/dashboard/cases`, but they still have to navigate manually to `/dashboard/vault` and retype the project name before they can review documents and compliance context.
- **Scope**: Add a per-case CTA from `/dashboard/cases` into `/dashboard/vault?q=<project>`, hydrate the Tech Vault search state from the URL, keep Vault search state synchronized back into the URL during editing/clearing, and add focused regression coverage for the handoff and URL-state behavior.
- **Expected impact**: Creates a high-value bridge from legal triage into evidence review, reducing friction in a core cross-workflow journey while staying inside existing Cases + Vault surfaces.
- **Risk (1-10)**: 4
- **Effort/size**: Medium

## Proposal L — Build a unified case evidence workspace across cases, protocols, and vault activity
- **Title**: Build a unified case evidence workspace across cases, protocols, and vault activity
- **Problem**: Users still have to reconcile case urgency, documentation completeness, and protocol activity across separate pages without one shared case workspace.
- **Scope**: Introduce a dedicated case workspace with evidence status, recent protocol history, next-action prompts, and cross-workflow rollups in one place.
- **Expected impact**: Strategically improves end-to-end product coherence for Swiss construction claims handling.
- **Risk (1-10)**: 7
- **Effort/size**: Large
