## Proposal S — Keep settings password feedback localized and self-clearing
- **Title**: Keep settings password feedback localized and self-clearing
- **Problem**: The settings password form stores the minimum-length error as a translated string in component state. If the user changes language after triggering the error, the message stays in the old language, and stale password feedback can persist while the user edits the field.
- **Scope**: Replace password feedback state with a structured translation/message model, render translation keys at display time, clear stale feedback when the password field changes, and add focused regression coverage for language-switch and edit-clear behavior.
- **Expected impact**: Makes account settings feedback more trustworthy across language changes and reduces confusion during password correction without expanding scope beyond one page.
- **Risk (1-10)**: 2
- **Effort/size**: Small

## Proposal M — Add a copy-link action for filtered case triage views
- **Title**: Add a copy-link action for filtered case triage views
- **Problem**: The cases dashboard already encodes filters in the URL, but there is still no explicit affordance for teams to copy a triage view and share it with collaborators.
- **Scope**: Add a copy-link CTA near the case filters, localized copied-state feedback, and targeted tests proving the shared link preserves the active triage view.
- **Expected impact**: Improves collaboration around urgent case review and turns existing URL-state behavior into a clearer user-facing workflow.
- **Risk (1-10)**: 4
- **Effort/size**: Medium

## Proposal L — Add per-case detail drill-down with linked protocol and evidence context
- **Title**: Add per-case detail drill-down with linked protocol and evidence context
- **Problem**: Users still need to move between the cases dashboard, protocol flow, and vault to understand the full context for a single case.
- **Scope**: Add a richer case detail surface that combines deadline status, checklist progress, linked protocols, and evidence context into one view.
- **Expected impact**: Deepens product usefulness for active compliance follow-up and reduces cross-page context switching.
- **Risk (1-10)**: 7
- **Effort/size**: Large
