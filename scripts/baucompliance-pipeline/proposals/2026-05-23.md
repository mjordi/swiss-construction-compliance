## Proposal S — Add a recovery link from protocol load failures back to Cases
- **Title:** Add a recovery link from protocol load failures back to Cases
- **Problem:** When the dashboard cannot hydrate a linked case, the current recovery path is mostly retry-only. Users who actually need to repair or reselect the case do not get a direct handoff back to the Cases workflow.
- **Scope:** Add a visible CTA from the dashboard linked-case error/stale state to the Cases page, prefilling the relevant case search when possible, and cover it with focused tests.
- **Expected impact:** Speeds up recovery when a linked case is missing or out of date and reduces dead-end frustration in the protocol flow.
- **Risk (1-10):** 3
- **Effort/size:** S

## Proposal M — Add in-place editing for existing cases
- **Title:** Add in-place editing for existing case metadata
- **Problem:** Users can create and delete cases but cannot correct project names, canton values, or contract/discovery dates without deleting the case and losing checklist continuity.
- **Scope:** Add an inline edit mode on the Cases page for one case at a time, reuse the existing date validation rules, persist updates to Supabase, surface localized update feedback, and refresh the derived timeline/status view after save.
- **Expected impact:** Removes destructive re-entry from a core workflow, preserves follow-up checklist progress, and makes the cases list substantially more usable for real legal/compliance work.
- **Risk (1-10):** 5
- **Effort/size:** M

## Proposal L — Add archive/unarchive lifecycle management across Cases and Vault
- **Title:** Add archive/unarchive lifecycle management across Cases and Vault
- **Problem:** The data model and Vault view already imply an archived lifecycle, but users still lack a first-class way to close and later restore completed matters without deletion.
- **Scope:** Introduce archive/unarchive actions, separate active vs archived case views more clearly, propagate the status through Cases and Vault UX, and add regression coverage for lifecycle transitions.
- **Expected impact:** Unlocks a strategically important non-destructive case lifecycle and sets up a cleaner long-term workspace model.
- **Risk (1-10):** 7
- **Effort/size:** L
