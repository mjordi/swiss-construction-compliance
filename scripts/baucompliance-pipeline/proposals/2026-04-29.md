## Proposal S — Require captured defect details for linked-case protocol finalization
- **Title:** Require captured defect details for linked-case protocol finalization
- **Problem:** A protocol linked to a compliance case can finalize with a blank defect description, yet the linked case may still be treated as if the defect was documented.
- **Scope:** Add a linked-case finalization guard, block finalization until the linked case is verified and a non-empty defect description is captured, and add localized inline guidance plus test coverage.
- **Expected impact:** Improves evidentiary integrity, reduces false checklist completion, and keeps case/protocol state aligned.
- **Risk:** 2/10
- **Effort/size:** Small (S)

## Proposal M — Add explicit retry/recovery states for dashboard write failures
- **Title:** Add explicit retry/recovery states for dashboard write failures
- **Problem:** Several dashboard writes still rely on generic alerts or optimistic state, which makes partial-write recovery hard for users during case and protocol workflows.
- **Scope:** Introduce inline error states and retry affordances for key case/protocol mutations, especially checklist toggles and protocol save follow-ups.
- **Expected impact:** Lowers abandonment during transient backend failures and improves operator trust in compliance workflows.
- **Risk:** 5/10
- **Effort/size:** Medium (M)
