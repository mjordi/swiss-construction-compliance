## Proposal S — Name legal reminder preset groups on deadline calculators

- **Target user and job-to-be-done:** Keyboard and screen-reader users selecting calendar reminders for legally sensitive Swiss construction deadlines.
- **Problem:** Reminder offset buttons expose `aria-pressed`, but the set of buttons is not programmatically grouped under “Calendar reminders,” so users navigating by controls can lose the context of what the day offsets affect.
- **Scope:** Add accessible group semantics (`role="group"` + `aria-labelledby`) around the reminder preset button groups on `components/ruegefrist-calculator.tsx` and `app/dashboard/deadlines/page.tsx`; add focused regression coverage in `__tests__/ruegefrist-calculator.test.tsx` and `__tests__/deadlines-share-link.test.tsx`.
- **Expected impact:** Improves confidence and accessibility when choosing reminders that feed copied links and ICS exports.
- **User-visible outcome:** Assistive technology announces the reminder preset controls as a named “Calendar reminders” group on both public and dashboard deadline calculators.
- **Competitive / best-product rationale:** Legal-compliance products should make deadline reminder controls understandable for all users, especially when the controls affect exported/shared legal deadlines.
- **Risk:** 2/10 — semantic markup only, covered by existing calculator tests.
- **Effort/size:** Small.
- **Phase-1 slice:** Name the existing reminder groups without changing reminder business logic, URL state, or ICS generation.

## Proposal M — Deadline reminder summary next to exported `.ics` actions

- **Target user and job-to-be-done:** Project admins exporting deadline reminders who need to verify which offsets will go into their calendar file.
- **Problem:** Users can toggle reminders, but the export buttons do not summarize the selected reminder offsets at the action point.
- **Scope:** Add a compact localized “Reminders: 14/7/1 days” summary near the public Ruegefrist and dashboard deadlines export/share controls, preserving existing URL/ICS behavior.
- **Expected impact:** Reduces accidental calendar exports with unintended reminder offsets.
- **User-visible outcome:** Users see selected reminder offsets immediately before exporting or sharing.
- **Competitive / best-product rationale:** Best legal deadline tools confirm reminder settings before producing audit/workflow artifacts.
- **Risk:** 4/10 — requires multi-language copy and careful layout on two surfaces.
- **Effort/size:** Medium.
- **Phase-1 slice:** Summary text only, no new presets or persistence changes.

## Proposal L — Unified deadline-to-action cockpit across calculators, cases, and protocol finalization

- **Target user and job-to-be-done:** Construction project teams managing a defect from deadline calculation through notice, evidence, protocol, and follow-up.
- **Problem:** Deadline calculations, cases, vault evidence, protocol finalization, and reminders remain spread across separate workflow surfaces.
- **Scope:** Create a cross-workflow cockpit/timeline showing calculated deadlines, notice readiness, evidence state, protocol state, reminders, and next legal action.
- **Expected impact:** Step-change toward an audit-ready compliance workspace.
- **User-visible outcome:** One view explains what is legally due next and what supporting artifacts are missing.
- **Competitive / best-product rationale:** Market-leading compliance products turn fragmented records into guided legal action plans.
- **Risk:** 8/10 — crosses cases, vault, protocol finalization, reminders, and open PR surfaces.
- **Effort/size:** Large/strategic.
- **Phase-1 slice:** Wait for #148/#149 to settle, then ship a pure timeline/view-model helper or a read-only dashboard card that consumes confirmed case action data.
