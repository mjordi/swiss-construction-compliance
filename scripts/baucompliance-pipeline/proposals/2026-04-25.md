## Proposal S — Actionable vault empty states for search and archived filters
- **Title**: Actionable vault empty states for search and archived filters
- **Problem**: When the vault returns zero cards, users only see generic fallback copy and have to guess whether they should clear the search, switch tabs, or create a new project.
- **Scope**: Add tab-aware empty-state messaging plus a single recovery CTA in `app/dashboard/vault/page.tsx`, backed by a small pure helper in `lib/vault.ts` and unit coverage in `__tests__/vault.test.ts`.
- **Expected impact**: Reduces dead ends in the vault and helps users recover from empty searches or archived/active mismatches without trial and error.
- **Risk (1-10)**: 2
- **Effort/size**: Small

## Proposal M — Localize the vault shell and relative update labels
- **Title**: Localize the vault shell and relative update labels
- **Problem**: The vault still exposes English-only headings, tab labels, placeholders, and relative-time strings inside an otherwise localized dashboard.

- **Expected impact**: Improves trust and consistency for German/French/Italian users on a core compliance evidence screen.
- **Risk (1-10)**: 4
- **Effort/size**: Medium

## Proposal L — Vault project detail panel with checklist and protocol context
- **Title**: Vault project detail panel with checklist and protocol context
- **Problem**: Vault cards show only top-level metadata, so users cannot review checklist progress or protocol history without bouncing to other dashboard surfaces.
- **Scope**: Add a project detail drawer/modal with checklist progress, protocol count context, and next-step guidance, likely touching vault layout, interaction state, and shared dashboard primitives.
- **Expected impact**: Turns the vault into a stronger operational review surface for evidence and follow-up work.
- **Risk (1-10)**: 7
- **Effort/size**: Large
