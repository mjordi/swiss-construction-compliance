# Research Proposals — 2026-04-26

## PM Insights (market + repo signal)
1. Swiss construction digitalization guidance continues to stress auditability and traceable documentation, so users now expect legal-context cues directly inside operational forms, not only in separate dashboard views.
2. Recent competitive messaging is still reliability-first (fewer promises, stronger continuity), which supports incremental workflow confidence improvements over broad feature expansion.
3. Last-7-day merged delivery in this repo remains S/M-only with no large merges, so today’s highest-probability shipping path is another tightly scoped workflow enhancement.

## Proposal S — Linked-case legal deadline context in protocol step 1
- **Size**: Small
- **Problem**: When users link a case in the wizard, they do not immediately see whether that case is urgent or under old-law immediate notice.
- **User value**: Improves legal clarity at the moment of drafting and reduces avoidable timing mistakes.
- **Implementation sketch**: After case selection, show compact context card with regime/status and notice deadline (or immediate-notice cue for old-law cases).
- **Risk**: Low

## Proposal M — Stale linked-case guard on draft restore
- **Size**: Medium
- **Problem**: Restored `selectedCaseId` can reference a deleted/inaccessible case, causing confusing linkage behavior.
- **User value**: Better data integrity and clearer recovery when drafts outlive their linked records.
- **Implementation sketch**: Validate restored case ID against current case list; if stale, show warning and one-click unlink.
- **Risk**: Medium-Low

## Proposal L — Protocol event timeline (immutable audit trail, phase 1)
- **Size**: Large
- **Problem**: Users cannot inspect a structured history of protocol lifecycle actions (drafted, signed, finalized, exported).
- **User value**: Higher trust and stronger compliance defensibility in disputes.
- **Implementation sketch**: Add append-only protocol events table, write events at key transitions, and expose a read-only timeline in case detail.
- **Risk**: High
