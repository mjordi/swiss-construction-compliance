## Proposal S — Auto-clear dashboard PDF download feedback

- **Problem:** After a dashboard protocol PDF download succeeds or fails, the localized feedback message remains visible indefinitely. That can make stale status look current after the user pauses on the success step.
- **Scope:** Add a guarded timeout that clears dashboard PDF download success/failure feedback after the existing short feedback window, and add focused tests for success and failure reset behavior.
- **Expected impact:** Keeps dashboard export status honest and aligned with existing share/copy feedback behavior elsewhere in the product.
- **Risk:** 2/10
- **Effort/size:** Small

## Proposal M — Preserve linked-case recovery context after refresh failures

- **Problem:** Linked-case recovery has strong existing coverage, but the UI could better distinguish stale cached case context from a failed fresh refresh when users are finalizing a protocol during intermittent connectivity.
- **Scope:** Review the linked-case loading state machine, add a clearer warning state for cached-vs-fresh case context, and extend tests for recovery after sequential success/failure/success requests.
- **Expected impact:** Improves trust in dashboard case handoffs under poor network conditions.
- **Risk:** 5/10
- **Effort/size:** Medium

## Proposal L — Guided protocol finalization review workspace

- **Problem:** The handover wizard still leaves final review steps spread across project data, defect input, signature, linked case context, and PDF export readiness.
- **Scope:** Design a dedicated final review workspace that summarizes evidence, linked case deadline context, required fields, signature status, and export readiness before finalization.
- **Expected impact:** High-value workflow clarity for construction handover compliance and fewer incomplete protocol exports.
- **Risk:** 8/10
- **Effort/size:** Large / strategic
