# Daily Product Improvement Proposals — 2026-07-02

## Research lens

**Primary user / job-to-be-done:** Swiss construction owners, contractors, and project teams triaging defect cases before drafting a notice, creating a protocol, or collecting evidence.

**Workflow moment where trust is lost:** In the cases list, users can see status chips and can open details for the legal next action, but the immediate row summary does not make the next legal action and deadline countdown visible before users choose between protocol, vault, reminder, or edit actions. This is the moment where a deadline-tracking product should reduce interpretation work.

**Best-in-market expectation:** A best-in-market Swiss construction-compliance product should make each case row an action dashboard: legal status, deadline urgency, evidence/protocol readiness, and next action should be visible without opening a disclosure. Users should not have to know which module hides the legally relevant next step.

## Proposal S — Promote next legal action and countdown in each case row

- **Target user and job-to-be-done:** Project admins triaging existing defect cases and deciding whether to draft a notice, create a protocol, export a reminder, or collect evidence.
- **Problem:** The list row currently exposes status chips, dates, protocol count, and workflow links, but the actual next legal action and countdown are hidden in the details disclosure. Users may click a workflow before seeing the action that should drive it.
- **Scope:** In `app/dashboard/cases/page.tsx`, add a compact visible action snapshot below the row header using existing `cases-next-legal-action`, `cases-deadline-countdown`, and `cases-reminder-readiness` data. Keep details content intact and add focused coverage in an existing cases test.
- **Expected impact:** Improves legal clarity and triage speed at the primary cases-list decision point.
- **User-visible outcome:** Each visible case immediately shows the next legal action and deadline countdown before the user opens details or jumps to protocol/vault.
- **Competitive / best-product rationale:** Leading compliance tools should surface action guidance in the primary list view, not only behind expandable detail sections.
- **Risk (1-10):** 2
- **Effort/size:** S / bounded phase-1 strategic slice

## Proposal M — Add a read-only case activity snapshot across protocol and evidence readiness

- **Target user and job-to-be-done:** Teams preparing a defect notice or meeting who need to know whether a case has protocols, evidence, checklist completion, and reminders ready.
- **Problem:** Case readiness is split across badges, details, vault links, and checklist state. The user lacks one compact activity snapshot that answers whether the case is audit-ready.
- **Scope:** Add a richer read-only snapshot component per case row combining protocol count, checklist progress, vault handoff, reminder readiness, and legal countdown. Avoid new persistence and new routes.
- **Expected impact:** Medium workflow value by turning the cases list into a readiness triage board.
- **User-visible outcome:** Case rows summarize legal and operational readiness in one cluster.
- **Competitive / best-product rationale:** Best products make audit readiness obvious at a glance across documents, reminders, and legal deadlines.
- **Risk (1-10):** 5
- **Effort/size:** M
- **Phase-1 slice:** Start with the S-sized visible legal action/countdown snapshot; add protocol/vault/evidence activity aggregation later after the dashboard handoff PR lands.

## Proposal L — Unified case command center with guided notice workflow

- **Target user and job-to-be-done:** Owners and contractors managing multiple legally sensitive defects who need a single place to see chronology, evidence, protocols, reminders, and next legal actions.
- **Problem:** BauCompliance has strong modules, but users still navigate between cases, vault, dashboard protocols, public calculators, and deadline tools to reconstruct the compliance story.
- **Scope:** Create a case detail/command-center route backed by the shared timeline model and persisted events, with guided notice drafting and exportable audit trail.
- **Expected impact:** High strategic differentiation: converts separate tools into a guided Swiss construction-compliance workspace.
- **User-visible outcome:** A case detail page shows the full legal/compliance chain and recommended next step.
- **Competitive / best-product rationale:** The category winner should provide audit-ready chronology and next-step guidance, not only calculators and lists.
- **Risk (1-10):** 8
- **Effort/size:** L
- **Phase-1 slice:** First extend row-level activity snapshots and shared event derivation; only then create a full command-center route.
