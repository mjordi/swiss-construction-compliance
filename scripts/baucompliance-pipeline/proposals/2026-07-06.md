# Daily Product Improvement Proposals — 2026-07-06

## User-and-market lens

Primary users are Swiss construction owners, contractors, and project teams moving between a defect case, the protocol handover engine, evidence vault, and deadline reminders. The workflow moment with the strongest product signal today is a cross-workflow handoff: a user clicks from a case/vault context into the dashboard protocol engine expecting the right legal context to carry over. A best-in-market Swiss construction-compliance product should make those handoffs auditable and self-explanatory, especially when a linked case has been archived, deleted, or is otherwise unavailable.

Recent pipeline decisions over the last 14 days show mostly user-facing workflow reliability, URL-state trust, reminder clarity, accessibility, and cross-workflow navigation improvements. Cases and deadline reminders have been optimized heavily; the dashboard protocol handoff is adjacent to those surfaces and is strategically useful without expanding into a broad cockpit.

## Proposal S — Explain missing linked-case handoffs in the protocol engine

- **Target user and job-to-be-done:** Project admins and contractors opening the dashboard protocol engine from a case/vault handoff to produce a legally reliable defect protocol.
- **Problem:** If the URL contains a `case=` handoff for a case that is no longer available, the dashboard removes the query param and silently shows a standalone protocol form. The user loses confidence because the product does not explain that the expected linked legal context could not be applied.
- **Scope:** Track when a requested handoff case is not found after cases load successfully, show the existing linked-case missing warning, let the user dismiss it, and clear the warning once a valid case is selected or a new handoff arrives.
- **Expected impact:** Higher trust in cross-workflow handoffs and fewer accidental standalone protocols when a case context is missing.
- **User-visible outcome:** Users see a clear "linked case not found" warning instead of a silent fallback.
- **Competitive / best-product rationale:** Best-in-market compliance tools should make legal-context loss explicit and auditable at handoff boundaries.
- **Risk (1-10):** 2
- **Effort/size:** S
- **Phase-1 slice:** This proposal is already the phase-1 slice.

## Proposal M — Add a protocol handoff context summary before finalization

- **Target user and job-to-be-done:** Project teams finalizing a protocol that was linked to a defect case.
- **Problem:** The dashboard shows linked-case context near the first step, but finalization does not summarize the source case and legal deadline context immediately before PDF/export.
- **Scope:** Reuse the selected case view model to render a compact linked-case summary in the final review/download area and include focused tests.
- **Expected impact:** Better legal artifact confidence at the final commitment point.
- **User-visible outcome:** Users can confirm the protocol is tied to the intended case before generating the PDF.
- **Competitive / best-product rationale:** Audit-ready tools should surface source context at export/finalization, not only during input.
- **Risk (1-10):** 4
- **Effort/size:** M
- **Phase-1 slice if M or L:** First render a read-only summary in the finalization step without changing PDF content.

## Proposal L — Unified compliance activity cockpit across cases, vault, protocols, and reminders

- **Target user and job-to-be-done:** Owners/contractors managing multiple projects who need one operational view of urgent notices, evidence readiness, protocol status, and reminders.
- **Problem:** Current workflow improvements are spread across separate pages; users still need to synthesize next actions across cases, vault, dashboard protocols, and deadlines.
- **Scope:** Create a new dashboard cockpit that consolidates per-project legal next action, evidence readiness, linked protocol status, and reminder/export state.
- **Expected impact:** Strong market differentiation and lower cognitive load for active construction-compliance operations.
- **User-visible outcome:** One prioritized activity view for legal compliance follow-up.
- **Competitive / best-product rationale:** A best-in-market product should make the next legally meaningful action obvious across all compliance artifacts.
- **Risk (1-10):** 8
- **Effort/size:** L
- **Phase-1 slice if M or L:** Start with a read-only activity card fed by existing case timeline and vault readiness contracts, then expand later.
