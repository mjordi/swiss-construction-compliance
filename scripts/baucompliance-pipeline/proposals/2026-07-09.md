# Daily Product Improvement Proposals — 2026-07-09

## Research lens

Primary users: Swiss construction owners, contractors, architects, and project teams who need to convert legal deadline calculations into audit-ready reminders, cases, protocols, and evidence without second-guessing whether the exported artifact reflects the chosen dates.

Workflow moment: after calculating an acceptance/defect deadline, a user copies a link or downloads a calendar file under time pressure. The product should make it obvious that the artifact was generated with the selected reminder settings, because missed or mismatched reminders can create legal-rights risk.

Best-in-market bar: a Swiss construction-compliance product should make every legal artifact action explicit, confirmable, localized, and aligned with the displayed calculation. The user should know when a calendar export succeeded and when to retry.

Recent 14-day pattern: recent decisions shipped many S/M slices around case handoffs, URL-state hygiene, reminder presets, readiness summaries, protocol review, and case calendar export idempotency. The portfolio is strongly user-facing, with repeated investment in deadlines/cases/protocols/evidence. Today should avoid another purely internal test or refactor patch and choose a small reliability slice only if it protects a real legal artifact workflow.

## Proposal S — Confirm dashboard deadline calendar exports

- **Target user / JTBD:** A contractor or owner reviewing calculated acceptance deadlines and downloading `.ics` reminders for their calendar.
- **Problem:** The dashboard deadline page confirms copied links but the `.ics` download button has no success/error feedback. Users cannot tell whether the legal calendar artifact was actually prepared, especially after changing reminder presets.
- **Scope:** Add localized temporary success/error feedback for dashboard `.ics` downloads, clear stale feedback when the source date/reminder state changes or resets, and cover the behavior with focused RTL tests.
- **Expected impact:** Improves confidence in a high-risk legal artifact action without changing deadline math.
- **User-visible outcome:** After clicking the calendar reminder download, the button label confirms that the calendar file was prepared or asks the user to retry.
- **Competitive / best-product rationale:** Best-in-market compliance tools should acknowledge legal artifact generation the same way they acknowledge sharing, exporting, and PDF actions.
- **Risk:** 2/10.
- **Effort/size:** S.
- **Phase-1 slice:** Implement feedback only for the dashboard deadlines `.ics` action; leave broader download analytics or persistent export history for later.

## Proposal M — Add a deadline-result action bridge into case tracking

- **Target user / JTBD:** A project lead who calculates acceptance deadlines and then wants to track the deadline as an ongoing case.
- **Problem:** Dashboard deadline results can be shared/exported but do not nudge the user into a tracked case workflow. Users may calculate a deadline and fail to operationalize it.
- **Scope:** Add a result CTA that opens the cases page with a prefilled, explicitly reviewed handoff for the relevant project/date fields, plus URL sanitization and tests.
- **Expected impact:** Turns a one-off calculator into an operational workflow bridge.
- **User-visible outcome:** Users can move from calculated deadlines to tracked cases without re-entering dates.
- **Competitive / best-product rationale:** Best products connect calculation, reminder, case tracking, evidence, and protocol work in one continuous flow.
- **Risk:** 5/10 because the dashboard acceptance-date semantics do not map perfectly to the current cases contract/discovery-date handoff.
- **Effort/size:** M.
- **Phase-1 slice:** Add an explicit review step or prefilled create form only after clarifying the date semantics, rather than silently mapping acceptance to discovery.

## Proposal L — Build an audit-ready legal artifact activity timeline

- **Target user / JTBD:** A construction team lead preparing for a dispute who needs a consolidated record of deadlines, calendar exports, protocols, evidence, and case checklist progress.
- **Problem:** Recent improvements make individual actions safer, but there is no single chronological activity record proving which legal artifacts were generated and when.
- **Scope:** Design and implement a dashboard timeline that records case-linked protocol finalization, evidence uploads, reminder exports, and checklist changes.
- **Expected impact:** Strategic trust and audit-readiness improvement.
- **User-visible outcome:** A user can review a consolidated audit trail per case/project.
- **Competitive / best-product rationale:** Legal compliance competitors differentiate on defensibility and audit trails, not only calculators.
- **Risk:** 8/10 due to persistence schema, event semantics, privacy, and migration requirements.
- **Effort/size:** L.
- **Phase-1 slice:** Start with a read-only UI summary sourced from existing confirmed case/protocol/checklist state before adding durable event storage.
