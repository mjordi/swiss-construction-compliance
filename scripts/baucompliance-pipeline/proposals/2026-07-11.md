## Proposal S — Confirm case deadline reminder exports inline

- **Target user and job-to-be-done:** Swiss project admins and contractors exporting a 60-day notice reminder from a tracked case while relying on the checklist as an audit/readiness ledger.
- **Problem:** The cases page currently downloads the `.ics` artifact and marks the checklist item complete, but gives no visible action feedback. If the browser blocks blob creation or export preparation fails, the user has no localized indication at the row action point.
- **Scope:** Add per-case temporary success/error feedback for `downloadCaseReminder`, clear stale feedback when a new export starts or source state changes, and keep the existing idempotent checklist completion behavior.
- **Expected impact:** Improves confidence in a legally relevant export flow without changing deadline math or persistence schema.
- **User-visible outcome:** After clicking “Export 60-day deadline reminder (.ics)”, the case details area temporarily shows that the calendar file was prepared, or that export failed.
- **Competitive / best-product rationale:** Legal deadline tools should make artifact preparation verifiable. This matches best-in-market expectations for audit-ready workflow feedback.
- **Risk (1-10):** 2
- **Effort/size:** S
- **Phase-1 slice:** This proposal is already the phase-1 slice; richer artifact history remains separate.

## Proposal M — Add a case row artifact-action summary

- **Target user and job-to-be-done:** Project teams triaging cases who need to see which legal artifacts have recently been prepared before deciding the next action.
- **Problem:** Case rows show readiness and checklist state, but not a consolidated “last prepared” artifact summary for reminder exports, protocols, and evidence handoffs.
- **Scope:** Render a compact read-only artifact summary using existing protocol counts and checklist state; avoid adding persistence fields.
- **Expected impact:** Helps users understand the operational state before opening details.
- **User-visible outcome:** Each case row would show a concise “Artifacts prepared” summary beside next action/countdown.
- **Competitive / best-product rationale:** Moves toward an audit cockpit by making legal artifacts visible at triage time.
- **Risk (1-10):** 5
- **Effort/size:** M
- **Phase-1 slice:** Start with read-only existing data only; no new event history or schema.

## Proposal L — Build an audit-ready legal artifact activity timeline

- **Target user and job-to-be-done:** Owners, contractors, and admins preparing evidence for disputes or handover review.
- **Problem:** Protocols, reminders, evidence, and checklist actions are spread across surfaces, making it hard to reconstruct what happened when.
- **Scope:** Add a unified activity timeline across cases, protocols, vault evidence, reminders, checklist changes, and exports.
- **Expected impact:** Major strategic auditability improvement.
- **User-visible outcome:** A case command-center timeline showing legal artifacts and next actions in one place.
- **Competitive / best-product rationale:** Best compliance products provide an audit trail, not just separate tools.
- **Risk (1-10):** 8
- **Effort/size:** L
- **Phase-1 slice:** Define a shared activity derivation helper from existing data before a new UI route.
