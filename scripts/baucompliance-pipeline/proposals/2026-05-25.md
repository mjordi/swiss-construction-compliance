## Proposal S — Make Vault project cards a real Cases handoff
- **Title:** Make Vault project cards a real Cases handoff
- **Problem:** `app/dashboard/vault/page.tsx` styles project cards as clickable (`cursor-pointer`) and shows a `MoreVertical` button, but neither surface does anything. Users must find the small "open in cases" CTA at the bottom of the card, which creates dead-control friction in a workflow that should quickly move from evidence browsing to case action.
- **Scope:** Make each Vault project card keyboard- and mouse-navigable to the same filtered Cases destination as the existing CTA, remove the dead menu affordance, preserve the existing CTA, and add focused regression tests for click and keyboard navigation.
- **Expected impact:** Improves cross-workflow navigation with minimal product risk by turning an already-visible but misleading card affordance into a fast path for action.
- **Risk (1-10):** 3
- **Effort/size:** Small

## Proposal M — Add archive / unarchive actions to Vault projects
- **Title:** Add archive / unarchive actions to Vault projects
- **Problem:** The Vault already has active and archived tabs, and `cases.status` already supports `archived`, but there is no user-facing way to move a project between those states. The Archived tab therefore behaves more like a passive view than an operational workflow.
- **Scope:** Add a lightweight project action menu or inline controls to archive and restore projects, update the Vault list state optimistically, and add targeted regression tests around tab movement and status persistence.
- **Expected impact:** Makes the Vault’s existing archived workflow actually usable for firms that want to clean up completed or dormant matters without losing access.
- **Risk (1-10):** 5
- **Effort/size:** Medium

## Proposal L — Build a dedicated case workspace route
- **Title:** Build a dedicated case workspace route
- **Problem:** Users still bounce across Cases, Vault, Deadlines, and Protocols to understand one claim’s urgency, evidence readiness, and next step. The product lacks a single per-case operating hub.
- **Scope:** Introduce a dedicated `/dashboard/cases/[id]` workspace that consolidates case status, follow-up checklist progress, protocol activity, deadline context, and evidence handoffs.
- **Expected impact:** Creates a clearer product backbone for Swiss construction defect handling and could become the primary cockpit for daily case work.
- **Risk (1-10):** 8
- **Effort/size:** Large
