## Small (S): Disable the success-step PDF download button while generation is pending

- **Problem:** The final handover success screen lets operators click the PDF download button repeatedly while the same generation flow is already in progress.
- **Scope:** Disable the download button during PDF generation, show the existing spinner affordance, and add a focused regression test.
- **Expected impact:** Prevents duplicate downloads and accidental repeated work with very low implementation risk.
- **Risk:** 2/10 — local button state only.
- **Effort/size:** S.

## Medium (M): Add localized dashboard PDF download feedback with stale-completion guards

- **Problem:** If handover PDF generation fails, the operator receives no visible feedback; if a new protocol is started while a download is pending, an older async completion can still update the success screen state.
- **Scope:** Add download feedback state for success and failure, guard `handleDownload` completions with a request id, clear stale feedback when starting a new protocol, disable the download button while pending, add localized strings, and cover success/failure/stale completion with focused tests.
- **Expected impact:** Improves trust in the final handover export step and prevents stale UI messages from appearing on a newer protocol.
- **Risk:** 4/10 — touches dashboard async UI state and PDF test mocks, but stays within one component and existing test file.
- **Effort/size:** M.

## Large / strategic (L): Case detail workspace for protocol, checklist, vault, and deadline state

- **Problem:** Case follow-up status, protocol completion, evidence vault links, and deadline risk remain spread across separate dashboard pages.
- **Scope:** Add a case detail route that aggregates protocol history, checklist progress, vault evidence, and deadline state, then link cases/vault/dashboard cards into that route.
- **Expected impact:** Strong workflow consolidation for compliance teams managing multiple defects and evidence streams.
- **Risk:** 8/10 — new route, multiple data sources, navigation semantics, and a broad test matrix.
- **Effort/size:** L.
