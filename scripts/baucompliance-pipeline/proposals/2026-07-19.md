# Daily Product Improvement Proposals — 2026-07-19

## User-and-market lens

Primary target users are Swiss construction owners, contractors, architects, and project admins using BauCompliance to triage defect cases under the 2026 OR/SIA deadline regime. The workflow moment is scanning the Cases ledger before deciding whether the next legally meaningful action is a protocol, reminder, notice draft, or evidence upload. A best-in-market Swiss construction-compliance product should make missing audit evidence obvious at scan time, not only after expanding details or discovering it during final protocol/Vault review.

Recent 14-day pipeline pattern: the portfolio has mostly shipped S-sized but legally meaningful slices around reminder exports, case/protocol handoffs, Vault audit context, linked protocol counts, archive/restore feedback, and checklist normalization. The recurring strategic backlog is a unified audit/activity cockpit. The safest next prerequisite is completing the scan-level Cases readiness picture with evidence completeness beside deadlines/reminders/protocols, without introducing a broad timeline model.

## PM insights

- Recent user-facing work has made legal states more visible, but the Cases summary still requires users to infer evidence readiness from a combined reminder string and expanded checklist controls.
- Evidence completeness is legally central: a deadline reminder without attached proof still leaves the case weak for later dispute/audit review.
- A small scan-level summary preserves daily shippability while moving toward the larger audit cockpit theme.
- There are no open PRs, so a narrow Cases-page slice has low branch-overlap risk.

## Proposal S — Show evidence readiness in the case action snapshot

- **Target user and job-to-be-done:** Swiss construction owners/admins scanning defect cases to decide which case needs proof attached before notice/protocol follow-up.
- **Problem:** The scan-level action snapshot already shows next legal action, deadline countdown, linked protocols, and reminder readiness, but evidence completeness is bundled inside reminder text and not visible as its own audit-critical dimension.
- **Scope:** Add a localized evidence-readiness cell to the existing `cases-action-snapshot-*` grid, using the shared normalized checklist state already computed for each case. Add focused RTL coverage.
- **Expected impact:** Users can see at a glance whether a case has proof attached before opening details or switching to the Vault.
- **User-visible outcome:** Case cards show a dedicated evidence readiness value (`Nachweise vollständig` / `Nachweise unvollständig`, localized) beside deadline/reminder/protocol context.
- **Competitive / best-product rationale:** Best-in-market compliance tools make audit-readiness state explicit at decision time; proof readiness should not be hidden behind checklist expansion.
- **Risk (1-10):** 2
- **Effort/size:** S
- **Phase-1 slice:** This proposal is the phase-1 slice: read-only scan-level display only; no checklist schema, Vault, filter, or deadline behavior changes.

## Proposal M — Add a first read-only case audit cockpit row

- **Target user and job-to-be-done:** Project admins reviewing the Cases ledger and wanting one compact row that combines legal action, deadline, reminder, protocol, evidence, and Vault state.
- **Problem:** Scan-level legal context is split across badges and an action snapshot, so users still assemble audit readiness mentally.
- **Scope:** Create a more structured audit row model from existing case timeline/checklist/protocol/Vault state, render priority and missing-artifact summary, and add tests for urgent and completed cases.
- **Expected impact:** Stronger movement toward a unified legal cockpit and faster triage for multi-project portfolios.
- **User-visible outcome:** Each case shows a compact audit summary of what is missing next and why.
- **Competitive / best-product rationale:** A premium compliance product should summarize legal readiness, not just expose raw data points.
- **Risk (1-10):** 5
- **Effort/size:** M
- **Phase-1 slice:** Start with read-only missing-artifact summary derived from existing state; defer routing, filters, and cross-page timeline aggregation.

## Proposal L — Build a unified audit-ready activity timeline

- **Target user and job-to-be-done:** Contractors/admins preparing for dispute review or internal audit across cases, protocols, Vault evidence, reminders, and deadlines.
- **Problem:** BauCompliance has the building blocks, but users cannot yet see a unified chronology of legal events and pending obligations.
- **Scope:** Introduce a derived activity timeline contract across cases, protocols, reminders, Vault evidence, and checklist changes, with dashboard/Cases/Vault read-only surfaces.
- **Expected impact:** Step-change product differentiation: BauCompliance becomes an audit cockpit, not just separate workflow pages.
- **User-visible outcome:** Users can answer “what happened, what proof exists, and what legal step is next?” from one activity view.
- **Competitive / best-product rationale:** The strongest Swiss construction-compliance products should provide audit-ready chronology and legal workflow traceability.
- **Risk (1-10):** 8
- **Effort/size:** L
- **Phase-1 slice:** Define one read-only derived timeline helper for a single case and render it behind one focused surface before persistence or cross-page rollout.
