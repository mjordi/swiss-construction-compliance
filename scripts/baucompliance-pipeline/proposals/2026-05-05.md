# Research Proposals — 2026-05-05

## PM Insights
- The last 7 days of merged work on `main` show a strong cadence of small, reviewable UX and reliability patches, with most daily pipeline PRs staying within a single page or helper plus focused tests.
- There are no currently open PRs, so overlap risk is low; the main calibration constraint today is staying within the repo’s recent small-change delivery envelope rather than avoiding active branch conflicts.
- The deadlines dashboard is a core user-facing workflow and recently gained share-link functionality, but its reminder controls still contain hardcoded German/UI text (`Kalender-Erinnerungen`, `Tage`, `Link`) and the share-link action gives no localized feedback when copying succeeds.

## Proposal S — Localize deadline reminder controls and share-link feedback
- **Title**: Localize deadline reminder controls and share-link feedback
- **Problem**: The deadlines dashboard still ships hardcoded reminder and share-link labels outside the translation system, so FR/IT/EN users see mixed-language UI in a primary workflow and receive no confirmation after copying a shared deadline link.
- **Scope**: Add locale keys for the reminder section label, reminder day suffix, share-link CTA, and copied-success feedback; update the deadlines page to use those keys and show a short copied state; extend focused tests for locale coverage and share-link behavior.
- **Expected impact**: Improves polish and trust in a high-traffic workflow, especially for multilingual teams using shared acceptance-date links.
- **Risk (1-10)**: 2
- **Effort/size**: Small

## Proposal M — Persist reminder presets in the shared deadline URL
- **Title**: Persist reminder presets in the shared deadline URL
- **Problem**: Shared deadline links restore the acceptance date but not the reminder offsets, so collaborators can open the same case context and still export different calendar reminder schedules.
- **Scope**: Encode selected reminder offsets into the URL, hydrate them on load, sanitize invalid values, and keep ICS export aligned with the shared configuration.
- **Expected impact**: Makes collaboration around deadline reminders more predictable and reduces accidental mismatch between shared review context and exported reminders.
- **Risk (1-10)**: 4
- **Effort/size**: Medium

## Proposal L — Add case-level deadline detail drawer with vault/protocol context
- **Title**: Add case-level deadline detail drawer with vault/protocol context
- **Problem**: Users reviewing deadlines still need to jump across the cases list, vault, and protocol surfaces to gather follow-up evidence and related artifacts for a single project.
- **Scope**: Add a richer deadline detail view that links a case’s reminder status, follow-up checklist, linked protocols, and vault evidence into one contextual panel.
- **Expected impact**: Deepens operational usefulness for teams managing many concurrent projects and compliance follow-ups.
- **Risk (1-10)**: 7
- **Effort/size**: Large
