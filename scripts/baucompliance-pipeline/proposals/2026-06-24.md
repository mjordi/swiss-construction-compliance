## Small (S) — Disable case action links while checklist persistence is pending

- **Problem:** Cases row checklist saves already lock checkboxes, edit/delete, and ICS export, but the row still exposes active "open in vault" and "create protocol" links while an optimistic checklist write is in flight. Users can navigate away before the pending persistence settles, making it unclear whether follow-up state was saved.
- **Scope:** In `app/dashboard/cases/page.tsx`, render non-navigable disabled action surfaces for the vault/protocol links while `isChecklistSaving` is true, and add focused RTL coverage proving pending rows have no active `href` for those actions and restore links after persistence resolves.
- **Expected impact:** Prevents cross-workflow navigation races during legal follow-up checklist saves and aligns cases with the product's existing pending-mutation lock discipline.
- **Risk (1-10):** 2
- **Effort/size:** Small

## Medium (M) — Add cases URL-param sanitization feedback for invalid filters

- **Problem:** The cases page parses invalid `regime`, `status`, and `sort` URL params to safe defaults but does not normalize the URL, which can leave copied/shareable links containing ignored state and make support/debugging harder.
- **Scope:** Sanitize invalid filter params on load, preserve valid sibling query params and unrelated params, update share-link tests, and add visible/implicit state consistency checks.
- **Expected impact:** More trustworthy shareable case views and fewer stale filter-state surprises.
- **Risk (1-10):** 4
- **Effort/size:** Medium

## Large / strategic (L) — Case activity timeline across protocols, vault, and reminders

- **Problem:** Project/case activity is spread across cases, protocols, vault documents, checklist actions, and reminder exports. Users lack one chronology to understand what has happened and what is still legally relevant.
- **Scope:** Phase a case activity timeline model and dashboard UI that combines protocols, evidence counts, checklist status, archive transitions, and reminder exports. Phase 1 could be read-only timeline derivation; later phases could add audit persistence.
- **Expected impact:** Strong strategic differentiation for Swiss construction compliance audits and handover defensibility.
- **Risk (1-10):** 7
- **Effort/size:** Large / strategic
