## PM insights

## Proposal S — Surface explicit failure feedback for case create/delete actions
- **Title:** Surface explicit failure feedback for case create/delete actions
- **Problem:** `app/dashboard/cases/page.tsx` currently closes the create form and refreshes the list even if Supabase insert/delete fails. Users can think a case was saved or removed when nothing persisted.
- **Scope:** Add mutation error state to the Cases page, keep the form open when create fails, show localized inline feedback for create and delete failures, and add focused regression tests.
- **Expected impact:** Improves trust in the core compliance workflow, prevents silent data loss confusion, and reduces support/debugging friction.
- **Risk (1-10):** 3
- **Effort/size:** S

## Proposal M — Add in-place editing for existing cases
- **Title:** Add in-place editing for existing case metadata
- **Problem:** Users can create and delete cases but cannot correct project names, canton values, or source dates without destructive re-entry.
- **Scope:** Add an edit mode for a case card or row, reuse existing validation for contract/discovery dates, persist changes, and refresh derived timeline/status state with focused tests.
- **Expected impact:** Meaningfully reduces churn in the primary case-management workflow and preserves protocol/checklist continuity when metadata needs correction.
- **Risk (1-10):** 6
- **Effort/size:** M

## Proposal L — Add archive/unarchive lifecycle management for resolved cases
- **Title:** Add archive/unarchive lifecycle management for resolved cases
- **Problem:** The Vault already models archived projects, but the inspected dashboard flow exposes no visible archive/restore action for cases, pushing users toward deletion or clutter.
- **Scope:** Introduce archive/unarchive controls, separate active vs archived visibility more clearly across Cases/Vault, and add regression coverage for lifecycle transitions.
- **Expected impact:** Unlocks a strategic non-destructive workflow for resolved matters and sets up a cleaner long-term workspace model.
- **Risk (1-10):** 7
- **Effort/size:** L
