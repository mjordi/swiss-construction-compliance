## Decisions

### APPROVED — Mark linked-case defect evidence even when the case checklist is empty

**User-impact note:** The target user is a Swiss contractor, owner, or project admin finalizing a linked acceptance protocol. The workflow moment is immediately after pressing **Finalize Protocol**, when the product writes legal evidence and should update the linked case readiness ledger. This is not incremental polish: if the case checklist starts empty, the product can otherwise fail to mark captured protocol evidence as documented, leaving the audit/readiness state misleading.

**Engineering task:** In `app/dashboard/page.tsx`, normalize the linked case checklist returned from Supabase and always update it with `defectDocumented: true` when a protocol finalizes for a linked case. Add focused coverage in `__tests__/dashboard-linked-case-retry.test.tsx` proving a linked protocol finalization persists `defectDocumented: true` even when the loaded checklist is `null`. Run `npm run test`, `npm run lint`, and `npm run build`.

### BACKLOG — Add a case-linked protocol activity snapshot on the case row

Valuable, visible, and strategically aligned, but broader than today's reliability fix because it requires a display contract for recent protocol state and possibly more copy/localization. The smallest credible future slice is a read-only protocol evidence summary using already-synchronized checklist/protocol state.

### BACKLOG — Build a unified audit-ready activity timeline across cases, protocols, vault evidence, and reminders

Strategically strongest, but too broad for one daily PR. The smallest future phase-1 is a read-only timeline from confirmed, already-synchronized events after today's protocol-to-checklist state gap is closed.
