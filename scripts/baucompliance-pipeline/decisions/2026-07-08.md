## Decisions

### APPROVED — Make case calendar reminder exports idempotent

**User-impact note:** Target users are Swiss construction owners, contractors, and project admins exporting a 60-day notice reminder from a case while relying on the follow-up checklist as an audit/readiness ledger. The workflow moment is repeating or re-downloading a case `.ics` reminder before a legal notice deadline. This is not incremental polish: repeated export should not undo the calendar-reminder completion state and make a legally relevant readiness marker appear incomplete.

**Engineering task:** In `app/dashboard/cases/page.tsx`, separate manual checklist toggles from export-triggered completion by adding an explicit checklist setter and making `downloadCaseReminder` set `calendarReminderExported` to `true`. Add focused coverage in `__tests__/cases-checklist-persistence.test.tsx` proving repeat `.ics` export keeps the checklist item complete and persists `calendarReminderExported: true`.

### BACKLOG — Add a case row “last exported reminder” confirmation state

Valuable follow-up because users should get explicit action confirmation near exports, but broader than today's correctness fix because it adds async UI feedback states and localized copy. The approved slice makes the underlying readiness state safe first.

### BACKLOG — Unified compliance activity cockpit across cases, protocols, vault evidence, and reminders

Strategically strongest, but too broad for one daily PR. The smallest credible future phase-1 is a read-only activity summary fed by already-stable case/vault/protocol readiness contracts after artifact actions are idempotent and feedback-safe.
