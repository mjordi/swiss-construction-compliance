## Decisions

### APPROVED — Canonicalize invalid vault tab query params while preserving project search

**User-impact note:** Target users are project teams and support users opening shared evidence-vault links during project evidence review or case follow-up. The workflow moment is landing on `/dashboard/vault` with a project search and tab state from a shared or stale link. This is not just incremental polish: a stale invalid `tab` value can leave a shareable evidence link claiming a different state than the UI actually applies, reducing trust in support/audit handoffs.

**Engineering task:** In `app/dashboard/vault/page.tsx`, normalize invalid owned `tab` query params whenever vault search params are consumed, preserving valid `q` and unrelated params. Add focused regression tests in `__tests__/vault-url-sync.test.tsx` proving invalid `tab` params are removed on initial load and on external query-param changes where parsed tab state is otherwise unchanged.

### BACKLOG — Associate Ruegefrist calculator visible date labels with inputs

Valuable and safe, but still an S-sized accessibility consistency patch. Backlog because the vault URL trust slice has clearer cross-workflow product value today.

### BACKLOG — Case activity timeline across protocols, vault, checklist, and reminders

Strategically compelling, but too large for one daily autonomous PR. The next credible phase-1 remains a pure timeline derivation helper with fixtures before UI/persistence.
