## Decisions

### APPROVED — Add next-action context to linked cases in the protocol wizard

**User-impact note:** Target users are Swiss construction project admins, owners, and contractors who open a case into the dashboard protocol workflow to document a defect or prepare an audit-ready protocol. The workflow moment is the handoff from a tracked case into protocol authoring, before the user writes defect text or finalizes evidence. This is not just incremental polish: it carries the legal next-action guidance from the shared case timeline into the protocol workflow, reducing the risk that users see a deadline date but miss the concrete legal action they must take.

**Engineering task:** In `app/dashboard/page.tsx`, derive the selected linked-case context from the shared `toComplianceCaseViewModel` contract instead of a local date/status-only calculation, and render the linked case's `nextAction` plus countdown/deadline context in the existing linked-case panel. Add focused dashboard regression coverage that a loaded linked case displays the next legal action and deadline context.

### BACKLOG — Show a case activity snapshot across case, protocol, evidence, and reminder readiness

Strong medium opportunity, but broader than today because it would touch row layout, protocol counts, vault readiness, checklist progress, and likely multiple existing cases tests. Keep its phase-1 as a read-only cases row snapshot once the dashboard context handoff is stable.

### BACKLOG — Unified compliance activity timeline and guided notice workspace

Strategically important but too large for one daily autonomous PR. The next prerequisite is a fixture-backed event model that combines protocol/vault/reminder events for a single case before UI or persistence.
