## Approved

### APPROVED — Surface next legal action on Vault project cards

**User-impact note:** Target users are Swiss construction owners, contractors, and project admins reviewing project evidence in the Technical Vault. The workflow moment is deciding whether stored proof is audit-ready or whether the case still needs a legal notice/protocol/reminder action. This is not just incremental polish: the Vault is a legal proof surface, and showing the next action plus deadline/countdown context prevents users from treating document storage as readiness when a legal workflow step remains open.

**Engineering task:** In `app/dashboard/vault/page.tsx`, carry `nextAction`, deadline/countdown, and status label from the existing shared case timeline into each `VaultProjectCard`, then render a compact read-only audit snapshot on each card. Add localized label keys in `locales/index.ts` and focused coverage in `__tests__/vault-follow-up-link.test.tsx` proving a Vault card displays the next legal action and deadline/countdown context without opening Cases. Run `npm run test`, `npm run lint`, and `npm run build`.
