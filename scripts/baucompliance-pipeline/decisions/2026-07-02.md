## Decisions

### APPROVED — Promote next legal action and countdown in each case row

**User-impact note:** Target users are Swiss construction project admins, owners, and contractors reviewing the cases list to decide the next operational/legal step. The workflow moment is list-level triage before opening details, exporting a reminder, creating a protocol, or collecting evidence. This is not just incremental polish: it makes the shared legal next action and deadline countdown visible at the primary decision point, reducing the chance that a user sees a status chip but misses the actual legal action.

**Engineering task:** In `app/dashboard/cases/page.tsx`, add a compact visible action snapshot under the row header for each case using existing `item.nextAction`, `item.deadlineCountdownLabel`, and reminder readiness values. Keep the existing details disclosure and actions unchanged. Add focused regression coverage in `__tests__/cases-url-sync.test.tsx` or a cases-focused test proving the visible row shows the next legal action and countdown without opening the details disclosure.
